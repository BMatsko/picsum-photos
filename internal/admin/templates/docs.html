Random image

GET /[width]/[height][.ext]

Returns a different random image on every request. Not cacheable.

ParameterTypeDescription
widthintegerOutput width in pixels
heightintegerOutput height in pixels
?grayscaleflagConvert to grayscale
?blur[=1–10]integer (optional)Gaussian blur; default intensity 5
?author=namestringFilter to photos by a specific author

Examples:

{{.RootURL}}/400/300
{{.RootURL}}/400/300.webp
{{.RootURL}}/400/300?grayscale
{{.RootURL}}/400/300?blur=3&grayscale
{{.RootURL}}/400/300?author=Jane

Seeded image

GET /seed/[seed]/[width]/[height][.ext]

Each (seed, tag) combination resolves to a specific photo and remembers it permanently. Passing a different tag creates an independent slot for that seed. Cached for 1 day.

ParameterTypeDescription
seedstringAny string — use an ID, slug, or UUID
?tag=namestringFilter initial resolution to photos with this tag. Omit or leave blank = any photo. Once resolved, the image is locked for that (seed, tag) pair.
?grayscaleflagConvert output to grayscale
?blur[=1–10]integerBlur the output

Examples:

{{.RootURL}}/seed/hello/400/300
{{.RootURL}}/seed/user-123/200/200?tag=portrait
{{.RootURL}}/seed/banner-home/1500/600?tag=landscape
{{.RootURL}}/seed/banner-home/1500/600?tag=landscape   ← always returns the same photo
{{.RootURL}}/seed/banner-home/1500/600?tag=urban       ← independent slot, different photo
Tag fallback: If the requested tag has no matching photos, the system falls back to any photo from the full library and stores the result as untagged.

Image by ID

GET /id/[id]/[width]/[height][.ext]

Fetch a specific photo by its numeric ID. Cached for 1 day.

{{.RootURL}}/id/1/400/300
{{.RootURL}}/id/1/400/300.avif
{{.RootURL}}/id/1/400/300?grayscale

Output formats

Append an extension to any image URL to request a specific format. Default is JPEG.

ExtensionMIME typeNotes
.jpg (default)image/jpegProgressive, optimised
.pngimage/pngLossless
.webpimage/webpModern lossy/lossless
.avifimage/avifAV1-based, best compression
.gifimage/gifAnimated GIFs preserve all frames
.tiffimage/tiffHigh-quality, lossless
.rawauto-detectedReturns the image in its stored format — GIF stays GIF, PNG stays PNG, etc. No re-encoding overhead.

{{.RootURL}}/id/2/800/600.webp

Upload API

All upload endpoints require a valid API key. Manage keys at API Keys.

Authentication
Authorization: Bearer pk_<your-key>
# or
?key=pk_<your-key>   (URL param, not recommended for uploads)
Upload — file attachment
POST /api/v1/upload
Content-Type: multipart/form-data

Fields:
  photo      file     required — JPEG, PNG, WebP, HEIC, HEIF, AVIF, TIFF, GIF
  author     string   required
  tags       string   optional — comma-separated, e.g. "nature, portrait"
  id         string   optional — auto-incremented if omitted
  url        string   optional — source label (defaults to filename)
curl -X POST {{.RootURL}}/api/v1/upload \
  -H "Authorization: Bearer pk_<key>" \
  -F "photo=@/path/to/image.jpg" \
  -F "author=Jane Doe" \
  -F "tags=nature,portrait"
Upload — remote URL
POST /api/v1/upload
Content-Type: application/json

{
  "photo_url": "https://example.com/photo.jpg",
  "author":    "Jane Doe",
  "tags":      "nature,portrait",
  "id":        "42"             // optional
}
# Also works as multipart with photo_url field instead of photo file
curl -X POST {{.RootURL}}/api/v1/upload \
  -H "Authorization: Bearer pk_<key>" \
  -F "photo_url=https://example.com/photo.jpg" \
  -F "author=Jane Doe"
Response — 201 Created
{
  "id":     "42",
  "author": "Jane Doe",
  "url":    "photo.jpg",
  "width":  4032,
  "height": 3024,
  "tags":   ["nature","portrait"]
}
List all images (authenticated)
GET /api/v1/images
Authorization: Bearer pk_<your-key>

Returns the full image list including tags — same as /v2/list but includes tags and requires auth.
